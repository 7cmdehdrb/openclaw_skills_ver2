# paper-summary-to-notion checklist

## A. 입력/중복
- [ ] canonical PDF 1개 확정
- [ ] sha256 fingerprint 계산
- [ ] 실행 락 확보(동일 fingerprint 동시 실행 방지)
- [ ] 부모 페이지(`논문`) 하위 중복 검사 수행
- [ ] 동일 `(제목+연도)` 또는 동일 fingerprint 존재 시 스킵 처리

## B. 메타/요약
- [ ] 페이지 제목이 `원문 제목 (연도)` 형식인지 확인 (축약 금지)
- [ ] `## 논문 정보`에 연도/저널/citation 수/APA/BibTeX 반영
- [ ] DOI가 있으면 표기, 없으면 `본문 확인 필요`로 표기
- [ ] `## 초록`이 원문 abstract의 직역 번역인지 확인(의역/추가/삭제 금지)
- [ ] `## 초록`은 quote가 아닌 일반 문단(paragraph)으로 작성
- [ ] `## 문제 상황` 섹션이 비어있지 않은지 확인
- [ ] `## 제안하는 방법`이 단순 열거가 아니라 "전체 흐름"이 보이도록 작성됨
- [ ] `## 제안하는 방법`에 최소 1개 설명 문단 + 단계형 흐름 불릿 + 마무리 연결 문장이 포함됨
- [ ] `## 제안하는 방법`에서 계층형 불릿 구조(최상위 단계 + 자식 불릿 상세)가 적용됨
- [ ] 최상위 불릿 4개 이상, 자식 불릿 총합 6개 이상
- [ ] `## 제안하는 방법`에 입력/출력, 절차(파이프라인), 차별점이 명시됨
- [ ] 필요 시 `제안하는 방법` 섹션만 2차 재요약하여 디테일 보강
- [ ] 표 사용 최소화(불릿 중심)

## C. 섹션/문법
- [ ] 섹션 순서 고정: 논문 정보 → 초록 → 문제 상황 → 제안하는 방법 → 결과 → 논문 이미지 → 원본 PDF
- [ ] 섹션과 섹션 사이 빈 문단 1줄로 시각적 구분이 들어갔는지 확인
- [ ] heading 블록 단일 라인
- [ ] 가짜 불릿(`-` 텍스트 노출) 없음
- [ ] heading 본문 침범 없음

## D. 이미지/PDF 첨부
- [ ] 이미지 추출 시 `max(width,height) >= 300` 필터 적용
- [ ] 필터 통과 이미지 전부 `## 논문 이미지` 섹션에 인라인 삽입
- [ ] 이미지 순서가 논문 원문 순서(페이지→인덱스)인지 확인
- [ ] 이미지 caption은 비워서 삽입(캡션 미사용)
- [ ] `## 원본 PDF` 섹션에 실제 file block 첨부
- [ ] `(아래에 ... 첨부)` 같은 지시 문장이 최종 페이지에 남아있지 않은지 확인

## E. 최종 QA/임시파일
- [ ] `scripts/validate_notion_page.py --page-id <id>` 통과
- [ ] 실패 시 rollback(in_trash) + 성공 보고 금지
- [ ] `source_fingerprint: ...` 문구가 본문 상단/중간에 노출되지 않았는지 확인
- [ ] 구조상 필요 시 페이지 최하단 메타 영역에만 배치
- [ ] 실제 사용된 PDF/이미지 경로만 임시파일 등록
- [ ] TTL 168시간(1주일) 등록
- [ ] 락 해제 확인

## F. 크론 배치 실행 시 추가
- [ ] canary 1건 선실행 통과 후 본 배치 시작
- [ ] 이전 작업 미종료 상태에서 다음 작업 시작 금지
