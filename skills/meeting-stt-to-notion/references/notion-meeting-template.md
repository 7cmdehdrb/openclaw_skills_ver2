# Notion Meeting Template (STT → Structured Notes)

Use this fixed structure and do not add extra sections.

## 1. 회의 개요
- 참석자
- 회의 성격
- 주요 주제

## 2. 주요 논의 내용
- 의제 1
  - 핵심 논의
  - 쟁점/판단 근거
- 의제 2
  - 핵심 논의
  - 쟁점/판단 근거
- 의제 3 ...

작성 구현 규칙:
- `의제`는 최상위 불릿 블록으로 생성
- 하위 항목은 반드시 해당 의제 블록의 자식 불릿(children)으로 생성
- `"  - 텍스트"` 형태의 들여쓰기 문자열로 대체하지 말 것

작성 규칙:
- 의제가 분리 가능하면 반드시 의제 단위로 묶기
- 추측 금지(불명확하면 단정하지 않기)
- 시간코드/분:초 정보는 본문에 쓰지 않기
- 의미 없는 보조 라벨(예: "의제별 상세") 쓰지 않기

## 3. 결정 사항
- [결정] 합의/방향/정책
- [결정] 우선순위/범위 확정

## 4. 지시 사항 / 액션 아이템
- 작업 1
  - 담당: OOO
  - 기한: YYYY-MM-DD 또는 미정
  - 우선순위: 높음/중간/낮음
  - 비고: 1줄 맥락
- 작업 2
  - 담당: OOO
  - 기한: YYYY-MM-DD 또는 미정
  - 우선순위: 높음/중간/낮음
  - 비고: 1줄 맥락

작성 규칙:
- 마크다운 테이블(`|`) 사용 금지
- 모든 항목은 Notion 불릿 + 자식 불릿으로 구조화
- 기존 "오픈 이슈/확인 필요" 성격 항목도 모두 여기로 편입
- 확인이 필요한 항목은 액션으로 변환(예: "OO 확인" 형태)
- 담당/기한 불명확 시 각각 "미지정"/"미정"으로 표기
